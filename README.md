# AI Daily Digest

把原始 AI / Tech 新闻流，变成一份真正可读、可追踪、可定时投递的日报。

`AI Daily Digest` 是一个面向 OpenClaw / 龙虾生态的技能。它不是简单地“抓 RSS + 让模型总结标题”，而是把整条日报生产流程拆成多个明确阶段：

- 抓取候选文章
- 过滤、去重、分片
- 核验更原始 / 更权威的来源
- 提取正文，而不是只依赖标题和简介
- 生成中文展开版摘要
- 渲染成适合聊天场景投递的最终日报
- 可选接入 cron，实现自动发送

这个技能适合：

- 想稳定追踪 AI / Tech 动态的人
- 想把“新闻整理”做成长期工作流的人
- 想在 OpenClaw 里运营一条日报流水线的人

它还有一个非常重要的优势：

> 它不是随便抓几个源，而是可以从 **90+ 信息源** 中筛选候选，重点覆盖高质量的 **AI / Tech 博客**、模型发布、工程更新与关键评论线索。

也就是说，这条技能的价值不只是“生成摘要”，而是先帮你站在一批优质信息源之上，再从里面选出真正值得读的内容。

---

## 这个技能能做什么

它可以：

- 从 RSS 和网页中生成 AI / Tech 日报
- 对候选内容去重和筛选
- 识别更原始、更权威的来源
- 尽量读取正文而不是只总结标题
- 生成中文结构化摘要
- 为每篇内容补充分类、关键词、推荐理由、评分
- 输出适合飞书 / 聊天消息的日报格式
- 手动运行，也可以通过 cron 定时运行
- 保留中间产物，方便排障和迭代

---

## 核心价值

很多所谓“AI 新闻机器人”只做到这一步：

- 拉 RSS
- 让模型总结
- 发出去

这样通常会带来几个问题：

- 标题党和转载文混进来
- 二手来源冒充一手来源
- 没读正文就直接总结
- 所有摘要都像同一个模板
- 看起来信息很多，但其实不好读

这个技能的独特之处在于：

> 它不是把所有问题都甩给最后一步模型，而是把质量控制前移到了抓取、核验、提取和编辑阶段。

所以它更像一条 **日报生产流水线**，而不是一个一次性的新闻摘要 prompt。

---

## 工作流概览

推荐按下面的阶段理解：

1. **Fetch**：抓取候选文章
2. **Plan**：过滤、去重、分片
3. **Scout**：形成初步信号与优先级
4. **Verifier**：寻找更原始、更权威来源
5. **Extractor**：提取正文
6. **Editor**：生成中文展开版摘要
7. **Render**：输出最终日报

这条流程天然适合“主任务 + 子任务”编排：

- 主任务负责调度、等待、收口、最终输出
- 子任务负责 Scout / Verifier / Extractor / Editor 这些重活

一句话：

> 子任务负责产物，主任务负责结果。

## 适合谁安装

### 对龙虾主人来说

如果你希望：

- 每天定时收到 AI / Tech 日报
- 不想自己翻几十个 RSS 和网页
- 又不想看那种“像模板一样的摘要”

那么这个技能适合你。

### 对接手技能的龙虾来说

如果你要接管这条流程，建议先确认：

- 日报要发到哪里
- 多久跑一次
- 覆盖哪些内容范围
- 模型和消息渠道是否已配好

建议先手动跑一版，再接 cron。

---

## 快速开始

推荐顺序：

1. 把技能放进 OpenClaw 工作区
2. 确认模型和消息渠道可用
3. 先手动跑一次日报
4. 检查 `output/` 里的中间产物和最终结果
5. 确认质量满意后，再接 cron 自动发送
